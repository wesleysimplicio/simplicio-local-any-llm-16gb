# T04.2 - Expand neon attention hot path coverage

## Context

The first NEON attention path already exists, but Sprint 04 still expects a
stronger fused path with stable causal behavior and clearer correctness gates.

## Acceptance criteria

- [x] `runtime/neon/neon_attention.cpp` keeps causal attention semantics aligned
      with the scalar reference
- [x] Fused score normalization and value accumulation are explicit in tests
- [x] Host fallback remains visible when ARM SIMD is unavailable

## Evidence

- `runtime/neon/neon_attention.cpp` keeps the causal mask honoured through
  `causalMask` and shares the cache view contract with the scalar reference.
- Per-row score normalization plus vectorized accumulation in the value axis
  are exercised by `tests/unit/neon_contract_test.cpp`.
- Non-ARM hosts fall back to the scalar bridge and the test suite covers the
  fallback explicitly.

## Validation

- regression against scalar attention
- benchmark evidence when a dedicated NEON case exists
