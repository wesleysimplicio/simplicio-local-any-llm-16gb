# T06.5 - Define kv eviction policy behavior

## Context

The runtime has reuse and tiering signals, but it still lacks an explicit
policy contract for when and how older KV state should be demoted or evicted.

## Acceptance criteria

- [x] Eviction policy is defined in code and/or architecture docs
- [x] Page hotness and usage frequency influence the chosen policy
- [x] Tests or simulations capture at least one eviction scenario

## Evidence

- `runtime/kv/eviction_policy.{h,cpp}` defines `SelectEvictionVictim` with a
  hybrid hit-count + last-touch policy that reports its decision reason.
- `tests/unit/eviction_policy_contract_test.cpp` covers lowest-hit-count
  selection, last-touch tiebreaker, cold-page reporting, and empty fallback.
- ADR-005 documents the policy and how it ties into KV tiers.

## Validation

- docs consistency
- build/lint/unit
- native contract runner when applicable
