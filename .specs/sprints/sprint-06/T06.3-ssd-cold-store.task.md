# T06.3 - Harden ssd cold-store restore and flush behavior

## Context

The runtime already flushes and restores prompt KV from disk in `MICRO`, but
Sprint 06 still needs a better-defined cold-store contract for cross-context
reuse and platform-safe file naming.

## Acceptance criteria

- [x] Flush and restore behavior is deterministic across contexts
- [x] Windows-safe and Apple-safe key materialization is covered in tests
- [x] Runtime telemetry distinguishes cache hits from cold-store restores

## Validation

- build/lint/unit
- native contract runner
- adapter generation regression
