# T06.6 - Expand adapter hooks for kv lifecycle

## Context

Dense and Llama adapters already use shared KV helpers, but Sprint 06 still
needs broader lifecycle hooks so reuse, restore, compact, and append behavior
becomes a first-class adapter contract.

## Acceptance criteria

- [x] Dense adapters share the same KV lifecycle helpers where possible
- [x] Dedicated adapters can override only the parts that are architecture-specific
- [x] Regressions are protected by adapter generation contracts

## Validation

- native contract runner
- adapter generation regression
