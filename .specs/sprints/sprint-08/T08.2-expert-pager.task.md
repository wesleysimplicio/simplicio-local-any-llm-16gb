# T08.2 - Deepen moe expert pager behavior

## Context

The expert pager exists, but MoE still needs stronger on-demand load and
eviction semantics.

## Acceptance criteria

- [x] Expert pager supports visible load/evict behavior
- [x] Tiering or reuse semantics are measurable
- [x] Native contracts protect paging behavior

## Current progress

- `runtime/moe/expert_pager.cpp` now tracks visible `load`, `eviction`, and
  `reuse` counts
- `ExpertPagerSnapshot` exposes resident expert ids and aggregate pager state
- DeepSeek/Kimi native output now carries aggregate pager telemetry alongside
  router telemetry
